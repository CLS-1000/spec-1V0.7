01 — WHAT THIS IS // product overview
Real-time signal monitoring for national security and geopolitical risk

SPEC-1 is an automated open-source research engine. It harvests signals from authoritative sources, filters them through a deterministic four-gate framework, and surfaces what matters — producing structured research records and daily written briefs without manual intervention.

The bottleneck is attention, not access.

One purpose: separate signal from noise in a high-volume information environment, and deliver what actually requires human attention.

02 — THE PROBLEM IT SOLVES // operational thesis

Anyone working in national security, geopolitical risk, or threat analysis faces the same daily reality: there is far more credible reporting than any human can process. Feeds from think tanks, investigative journalists, and defense research organizations produce dozens of new pieces every day. Most are not actionable. A small fraction are. The challenge is not access to information — it is triage.

SPEC-1 automates that triage. It monitors a curated set of high-credibility sources continuously, evaluates each new signal through a structured scoring framework, and surfaces only those that clear every threshold. Signals that pass are handed to an investigation and verification workflow powered by Claude. The result is a compressed, high-confidence set of research records and a daily written brief — ready for human review without the noise.

"The system handles the volume. The analyst handles the judgment."
03 — PIPELINE // 7 stages · fault-tolerant · daily cron 06:00 PT

Seven sequential stages. Each transforms the data and gates what moves forward. Any single failure is logged and skipped — the cycle never halts on a single error.

04 — DOMAIN ADAPTERS // cls_osint/adapters/

The core pipeline is domain-agnostic. Domain-specific analysis is handled by adapter modules that feed into the same scoring and persistence infrastructure.

FARA
cls_osint/adapters/fara  ·  22 tests
DOJ public database filings. Cross-references foreign agent registrations against congressional activity and known influence networks. Surfaces registrant-principal relationships as scored signals.
CONGRESSIONAL
cls_osint/adapters/congressional  ·  35 tests
Member financial disclosures via fallback chain: Quiver Quantitative → Capitol Trades → House eFD. Flags trades in defense, cybersecurity, and energy sectors for conflict-of-interest analysis.
NARRATIVE
cls_osint/adapters/narrative
Psychological operations detection. TF-IDF cosine similarity clustering, entity anomalies, publication silence patterns. Surfaces narrative seeding, astroturfing, and information suppression as scored signals.
PERSISTENCE
cls_db.dual_write
Atomic dual-write layer. Every store write keeps JSONL and SQLite in sync. JSONL is the immutable ground truth; SQLite is the query layer for historical analysis and downstream access.
06 — DAILY RESEARCH BRIEF // Claude Sonnet  ·  rule-based fallback if API unavailable

After each cycle, a briefing generator submits scored research records to Claude Sonnet with a professional editor prompt — precise, sourced, no speculation beyond the evidence. Every claim traces back to a scored signal. If the API call fails, a structured fallback brief is generated from raw record data. The cycle never produces nothing.

  • Executive summary of cycle findings
  • Elevated signals requiring immediate attention
  • Domain briefings — cyber and geopolitical separately
  • Story leads for further investigation
  • Watch list items for ongoing monitoring

11 — WHY THIS MATTERS
The system handles the volume. The analyst handles the judgment.

The bottleneck in analysis work is not information — it is attention. SPEC-1 is designed to protect analyst attention by doing the triage work that does not require human judgment: fetching, cleaning, filtering, and structuring.

What reaches the human is already scored, investigated, verified, and written up. The human role is then what it should be: evaluating the findings, directing follow-on research, and making decisions.

That division of labor is the point.

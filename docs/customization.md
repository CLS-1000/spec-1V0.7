## Add a new RSS source

RSS sources are declared in `src/cls_osint/sources.py`. Each entry is a dict with
at minimum a `url` and a `source_type`.

```python
# src/cls_osint/sources.py
SOURCES = [
    # ... existing sources ...
    {
        "name": "My Custom Feed",
        "url": "https://example.com/feed.rss",
        "source_type": "rss",
        "credibility": 0.7,    # 0.0–1.0; contributes to gate 1
        "tags": ["policy", "finance"],
    },
]
```

After adding the source, run a cycle:

```bash
make cycle
```

The 4-gate scorer will evaluate each new signal automatically. Raise `credibility`
if signals from this source are consistently underscoring gate 1.

## Add a new signal adapter (FARA-style)

Adapters live under `src/cls_osint/adapters/`. Each adapter must expose a
`collect() -> list[OSINTRecord]` function.

**Step 1 — Create the adapter:**

```python
# src/cls_osint/adapters/my_source.py
from __future__ import annotations
from cls_osint.schemas import OSINTRecord
import requests

def collect() -> list[OSINTRecord]:
    resp = requests.get("https://api.example.com/data", timeout=15)
    resp.raise_for_status()
    return [
        OSINTRecord(
            record_id=f"my-{item['id']}",
            source_type="MY_SOURCE",
            source_name="My Source API",
            content=item["text"],
            url=item.get("url", ""),
            collected_at=item.get("date", ""),
        )
        for item in resp.json()
    ]
```

**Step 2 — Register the adapter in the pipeline:**

```python
# src/cls_osint/pipeline.py  (add to the _ADAPTERS list)
from cls_osint.adapters import my_source

_ADAPTERS = [
    # ... existing ...
    my_source,
]
```

**Step 3 — Add tests:**

```python
# tests/test_my_source.py
from unittest.mock import patch
from cls_osint.adapters.my_source import collect

def test_collect_returns_records(tmp_path):
    mock_data = [{"id": "1", "text": "Test signal", "url": "https://example.com"}]
    with patch("requests.get") as mock_get:
        mock_get.return_value.json.return_value = mock_data
        mock_get.return_value.raise_for_status = lambda: None
        records = collect()
    assert len(records) == 1
    assert records[0].source_type == "MY_SOURCE"
```

## Modify scoring thresholds

The 4-gate pipeline thresholds are defined in `src/spec1_core/signal/gates.py`:

```python
# src/spec1_core/signal/gates.py
GATE_THRESHOLD   = 0.40   # minimum score per gate (raise to be more strict)
VOLUME_MIN_WORDS = 50     # gate 2 — minimum word count
VELOCITY_MAX_AGE = 48     # gate 3 — maximum signal age in hours
```

**Calibration workflow:** Rather than editing thresholds blindly, use the
calibration tool to diagnose drift first:

```bash
make calibration
# Produces: generated/calibration_report.md + generated/calibration_proposals.md
```

The report shows per-gate accuracy bucketed by confidence score. Adjust
thresholds only after reviewing the report.

> **Important:** Threshold changes affect all future signals. Document every
> change in `CHANGELOG.md` with the calibration evidence that justified it.

## Change analyst weights

Analyst credibility weights are stored in the `AnalystRecord` registry at
`src/spec1_core/analysts/`. Each analyst has a `credibility_score` (0.0–1.0)
that scales the gate 1 score for signals attributed to them.

```python
# src/spec1_core/analysts/registry.py
ANALYSTS = [
    AnalystRecord(
        analyst_id="analyst-001",
        name="Jane Smith",
        affiliation="RAND Corporation",
        domains=["defense", "strategy"],
        credibility_score=0.85,   # ← adjust here
    ),
    # ...
]
```

After adjusting weights, re-run the calibration tool to validate the impact:

```bash
make calibration
```

## Customize brief templates

Briefs are generated by `src/spec1_core/briefing/`. Two paths:

**1. Claude Sonnet path (requires `ANTHROPIC_API_KEY`):** The prompt template is
   at `src/spec1_core/core/prompts/brief.md`. Edit this file to change the
   structure, focus, or tone of Claude-generated briefs.

**2. Rule-based fallback path:** The rule-based producer is in
   `src/cls_world_brief/producer.py`. Edit `produce_brief()` to change the
   section structure, lead-sentence logic, or source attribution.

**Brief modes** (`--mode` flag for `generate_brief`):

| Mode | Description |
|------|-------------|
| `standard` | General OSINT brief (default) |
| `geopolitics` | Foreign policy and state actor focus |
| `legislative` | Congressional and regulatory focus |

```bash
PYTHONPATH=src python -m spec1_core.tools.generate_brief \
    --mode geopolitics \
    --out-dir generated/briefs
```

## Integrate with external services

### Slack webhook

Post the latest brief to a Slack channel after each cycle:

```python
# scripts/post_to_slack.py
import os, requests
from pathlib import Path

webhook = os.environ["SLACK_WEBHOOK_URL"]
brief = Path("generated/briefs/spec1_brief_latest.md").read_text()
# Truncate to Slack's 3000-char limit
payload = {"text": f"*SPEC-1 Brief*\n```{brief[:2800]}```"}
requests.post(webhook, json=payload)
```

Add to Makefile:

```makefile
post-slack:
	PYTHONPATH=src python scripts/post_to_slack.py
```

### Email digest (SMTP)

```python
# scripts/send_digest.py
import os, smtplib
from email.mime.text import MIMEText
from pathlib import Path

brief = Path("generated/briefs/spec1_brief_latest.md").read_text()
msg = MIMEText(brief)
msg["Subject"] = "SPEC-1 Daily Intelligence Brief"
msg["From"] = os.environ["SMTP_FROM"]
msg["To"] = os.environ["SMTP_TO"]

with smtplib.SMTP_SSL(os.environ["SMTP_HOST"], 465) as s:
    s.login(os.environ["SMTP_USER"], os.environ["SMTP_PASS"])
    s.send_message(msg)
```

### GitHub Actions trigger

```yaml
# .github/workflows/daily-cycle.yml
name: Daily Intelligence Cycle
on:
  schedule:
    - cron: "0 6 * * *"   # 06:00 UTC
jobs:
  cycle:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.11" }
      - run: pip install -e ".[dev]"
      - run: PYTHONPATH=src python -m spec1_core.app.cycle
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
      - run: PYTHONPATH=src python -m spec1_core.tools.generate_brief
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

### Webhook on new records (POST to external URL)

```python
# src/spec1_core/intelligence/store.py — extend JsonlStore.write()
import requests

def write(self, record: dict) -> dict:
    entry = super().write(record)  # existing logic
    webhook = os.environ.get("SPEC1_WEBHOOK_URL")
    if webhook:
        try:
            requests.post(webhook, json=entry, timeout=5)
        except Exception:
            pass  # non-fatal
    return entry
```
